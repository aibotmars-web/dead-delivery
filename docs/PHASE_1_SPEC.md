## Scope Rules（嚴格範圍控制）

### Phase 1 只做這些，多一個都不加：
```
✅ 送餐核心（1地圖、3餐廳、5客戶）
✅ 隨機事件（Layer 1 抽卡 + Layer 2 城市事件）
✅ 裝備升級（機車+外送包，各 3 階）
✅ 成就系統（10 個）
✅ 每日任務（3 個）+ 連登 7 天
✅ 廣告（Banner + Interstitial + Rewarded）
✅ Web 部署
```

### Phase 1 絕對不做：
```
❌ 股票投資 → Phase 2
❌ 房間佈置 → Phase 3
❌ 寵物系統 → Phase 3
❌ 主線劇情 → Phase 2
❌ AI 新聞卡 → Phase 3
❌ 多點連送 → Phase 2
❌ 人際關係 → Phase 2
❌ 機車外觀自訂 → Phase 2
❌ 季節活動 → Phase 2+
❌ App 版本 → Phase 4
```

## 1. Map System（地圖系統）

### 1.1 Map Layout
```
地圖大小：50 x 50 tiles（16px each = 800x800 px world）
顯示縮放：2x（一個 tile 顯示為 32px）
可視範圍：約 12 x 20 tiles（手機直式畫面）

地圖結構（Phase 1 只有 1 張地圖）：

    ┌───────────────────────────────┐
    │        住宅區                  │
    │   [公寓A] [公寓B]              │
    │      │      │                  │
    │──────┼──────┼─────────────────│
    │      │      │                  │
    │   [你家]    │    [客戶C]       │
    │      │      │       │          │
    │══════╪══════╪═══════╪═════════│ ← 主要道路
    │      │      │       │          │
    │  [餐廳1]  [餐廳2]  │          │
    │      │      │       │          │
    │──────┼──────┼───────┼─────────│
    │      │      │       │          │
    │          夜市區                │
    │   [攤位] [攤位] [餐廳3]        │
    │                                │
    │   [客戶D]        [客戶E]       │
    │                                │
    └───────────────────────────────┘

固定地標：
  🏠 你家（起點/終點）
  🏪 餐廳 x3（取餐點）
  🏢 客戶 x5（送餐點）
  🚦 紅綠燈 x6
  🏪 便利商店 x1（買東西恢復體力，Phase 2 用）

道路特色：
  - 主要道路（寬，車多）
  - 巷弄（窄，捷徑但有障礙）
  - 夜市街道（人多，速度慢但有特殊事件）
```

### 1.2 Navigation
```
操作方式：點擊地圖上的目的地
移動方式：角色自動沿道路移動（A* pathfinding）
移動速度：
  - 走路：1 tile/秒
  - 騎車：3 tiles/秒（Lv.1）→ 4（Lv.2）→ 5（Lv.3）
  - 下雨時：速度 x0.7
  - 夜市區：速度 x0.5（人多）

鏡頭：永遠跟隨角色，角色在畫面中央
```

## 2. Order System（訂單系統）

### 2.1 Order Generation
```
每次可接 1 單（Phase 1 不支援多單）

訂單資訊：
  - 餐廳名稱 + 位置
  - 客戶名稱 + 位置  
  - 餐點名稱（顯示用，不影響玩法）
  - 報酬金額：$30-$150
  - 時限：3-8 分鐘
  - 距離標示（近/中/遠）

報酬計算：
  base = 距離 x $10 + $20
  準時 = base x 1.0
  提前 = base x 1.2 + 可能有小費
  超時 < 2min = base x 0.8
  超時 < 5min = base x 0.5
  超時 > 5min = 訂單取消，$0

小費（隨機）：
  - 無小費：60%
  - $10-30：25%
  - $50-100：10%
  - $200+：5%（稀有好客人）
```

### 2.2 Delivery Flow（送餐流程）
```
State Machine：

  IDLE（待機）
    → 打開手機 → 看訂單列表
    → 點擊「接單」
    
  PICKUP（取餐）
    → 地圖顯示餐廳位置標記
    → 點擊餐廳 or 自動導航
    → 到達餐廳 → 觸發取餐事件
    → 等待 2-10 秒（餐準備時間，隨機）
    → 取餐完成 → 計時開始
    
  DELIVERING（送餐中）
    → 地圖顯示客戶位置標記
    → 點擊客戶位置 or 自動導航
    → 途中可能觸發隨機事件
    → 到達客戶位置 → 觸發送餐事件
    
  DELIVERED（送達）
    → 顯示報酬結算
    → 抽一張機會/命運卡
    → 回到 IDLE
    
  FAILED（失敗）
    → 超時太久 / 餐全毀
    → 無報酬
    → 扣評分
    → 回到 IDLE
```

### 2.3 Order Variety（Phase 1 共 15 種組合）
```
餐廳 3 家：
  🍜 阿嬤滷肉飯
  🧋 茶之道手搖
  🍗 爆漿雞排王

客戶 5 位：
  👨 小陳（3樓，友善，常給小費）
  👩 林小姐（12樓沒電梯，趕時間）
  👴 王伯伯（1樓，但地址常寫錯）
  👧 學生妹（宿舍，訊號差常找不到人）
  😡 奧客張（態度差，但單價高）

每個客戶有自己的：
  - 小費傾向
  - 特殊事件機率
  - 樓層
  - 性格（影響送餐事件）
```

## 3. Random Event System（隨機事件系統）

### 3.1 Layer 1: Cards（抽卡系統）
```
觸發時機：每完成一單送餐
動畫：卡片從底部飛入 → 翻轉 → 顯示

Phase 1 卡池（20張）：

🟡 機會卡 x12（60%）：
  01. 小費加碼 → +$50
  02. 連續綠燈 → 下一單移動速度 +20%
  03. 餐廳秒出餐 → 下一單不用等
  04. 撿到零件 → 機車經驗值 +10
  05. 路上找到 $100
  06. 客人五星好評 → 評分 +0.1
  07. 順風 → 移動速度 +10% 持續 2 分鐘
  08. 手機滿電 → 導航加強（小地圖變大）
  09. 外送包完好 → 保溫加成，不怕餐涼
  10. 偶遇同行 → 獲得情報（下一單客戶提示）
  11. 陽光普照 → 心情好，所有效果 +5%
  12. 意外打折 → 下次升級裝備打 8 折

🔴 命運卡 x7（35%）：
  13. 突然下雨 → 速度 -20% 持續 3 分鐘
  14. 被開罰單 → -$300
  15. 手機快沒電 → 小地圖消失 2 分鐘
  16. 輪胎漏氣 → 速度 -30% 直到回家修
  17. 外送包拉鍊壞 → 餐點品質下降
  18. 差評 → 評分 -0.1
  19. 塞車 → 下一單途中強制等待 30 秒

🟣 稀有卡 x1（5%）：
  20. 外送之神降臨 → 下 3 單報酬翻倍
```

### 3.2 Layer 2: City Events（城市事件）
```
觸發時機：騎車移動中，每 10 秒判定一次，15% 機率觸發
顯示方式：彈出選擇框

Phase 1 城市事件（10 個）：

路上事件（5個）：
  E01. 紅燈
    「前方紅燈，要等嗎？」
    A: 等（-15秒）
    B: 闖（30%機率罰單 -$600，70%安全通過）

  E02. 三寶切車
    「有人突然切入你前方！」
    A: 急煞（餐可能晃出 20%機率，-3秒）
    B: 閃避（50%成功省時，50%摔車 -$200 -30秒）

  E03. 下雨
    「開始下雨了」
    A: 穿雨衣（速度 -15%，但安全）
    B: 硬騎（餐品質 -20%，可能滑倒 10%）

  E04. 施工改道
    「前方道路施工」
    [自動改道，+20秒]
    （無選擇，純時間懲罰）

  E05. 路上有狗
    「一隻狗衝向你！」
    A: 停下來等（-10秒）
    B: 加速閃過（20%被追更久 -20秒，80%成功）

取餐事件（2個）：
  E06. 餐還沒好
    「老闆說再等一下...」
    A: 耐心等（-30秒，店家好感+）
    B: 催餐（-15秒，但 30%機率餐做錯要重來）

  E07. 飲料忘了做
    「啊！忘了做飲料！」
    [自動等待 -20秒]

送餐事件（3個）：
  E08. 客人不接電話
    「打了 3 通都沒接...」
    A: 繼續打（-30秒，最終接到）
    B: 放門口（省時，但 40%差評）

  E09. 樓層挑戰
    「客人在 X 樓，沒電梯」
    A: 爬樓梯（-15秒 x 樓層數）
    B: 叫客人下來（50%願意，50%差評）
    C: 看廣告叫電梯（Rewarded Ad）

  E10. 備用湯匙
    「客人問：有湯匙嗎？」
    有備品包裝備 → 「有！」→ 小費 +$50
    沒有 → 「沒有...」→ 評分 -0.05
```

## 6. Daily System（每日系統）

### 6.1 Daily Missions（每日任務，每天刷新 3 個）
```
任務池（隨機抽 3 個）：
  - 完成 3/5/8 單送餐
  - 準時送達 2/3/5 次
  - 累計賺 $300/$500/$1000
  - 在下雨中完成 1 單
  - 抽到 1 張稀有卡
  - 不被開罰單完成 3 單
  - 獲得 1 次五星好評

獎勵：每完成 1 個 → $100，全完成 → 額外 $500
```

### 6.2 Login Streak（連續登入）
```
Day 1: $200
Day 2: $300
Day 3: $500 + 機會卡 x1
Day 4: $500
Day 5: $800
Day 6: $800
Day 7: $1,500 + 稀有機會卡 x1

斷了重來。
登入 = 打開遊戲 + 完成至少 1 單。
```

## 8. Data Model（資料模型）

### 8.1 Player
```dart
class Player {
  String name;
  int money;           // 持有金額
  double rating;       // 外送評分 1.0-5.0
  int totalDeliveries; // 累計送餐數
  int todayDeliveries; // 今日送餐數
  int todayEarnings;   // 今日收入
  int loginStreak;     // 連續登入天數
  DateTime lastLoginDate;
  Equipment scooter;   // 機車
  Equipment bag;       // 外送包
  List<String> achievements; // 已解鎖成就 ID
  List<DailyMission> dailyMissions; // 今日任務
  int dayNumber;       // 遊戲天數（第幾天）
}
```

### 8.2 Order
```dart
class Order {
  String id;
  Restaurant restaurant;
  Customer customer;
  String foodName;
  int reward;          // 基本報酬
  int timeLimit;       // 時限（秒）
  int tipAmount;       // 小費
  OrderStatus status;  // idle/pickup/delivering/delivered/failed
  DateTime? startTime;
}
```

### 8.3 Equipment
```dart
class Equipment {
  String id;
  String name;
  int level;           // 1-3
  int upgradeCost;     // 升級費用
  Map<String, double> stats; // speed, stability, insulation...
}
```

### 8.4 EventCard
```dart
class EventCard {
  String id;
  CardType type;       // chance / fate / rare
  String title;
  String description;
  String iconEmoji;
  Map<String, dynamic> effect; // money, speed, rating...
}
```

### 8.5 CityEvent
```dart
class CityEvent {
  String id;
  String title;
  String description;
  List<EventChoice> choices;
  EventTrigger trigger; // road / pickup / delivery
}

class EventChoice {
  String label;
  String effectDescription;
  Map<String, dynamic> effect;
  double successRate;    // 1.0 = guaranteed
}
```

## 11. Success Metrics（驗證指標）

```
上線 1 週後評估：

✅ 繼續做 Phase 2 的條件（符合任一）：
  - Day 1 Retention > 40%
```
